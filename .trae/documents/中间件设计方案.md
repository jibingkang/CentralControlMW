# 中间件设计方案

## 项目概述
- **项目名称**: CentralControlMW (中间件)
- **功能**: 接收终端设备按键回调，根据设备ID查找对应转换码，通过TCP客户端发送
- **技术栈**: Python 3.x + FastAPI + MySQL + TCP Client
- **数据库**: MySQL, 数据库名: `db_centralcontrolmw`
- **连接信息**:
  - 地址: 192.168.1.200
  - 端口: 3306
  - 账号: eslsdb
  - 密码: eslsdb@pi

## 数据库设计

### 表: device_mapping (设备ID与转换码绑定表)
```sql
CREATE TABLE device_mapping (
    id INT AUTO_INCREMENT PRIMARY KEY,
    device_id VARCHAR(50) NOT NULL UNIQUE COMMENT '设备ID，如 92.95.40.81',
    convert_code VARCHAR(100) NOT NULL COMMENT '转换码，如 0E 99 EE 22 25',
    description VARCHAR(200) DEFAULT '' COMMENT '描述',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 表: system_config (系统配置表)
```sql
CREATE TABLE system_config (
    id INT AUTO_INCREMENT PRIMARY KEY,
    config_key VARCHAR(50) NOT NULL UNIQUE,
    config_value VARCHAR(200) NOT NULL,
    description VARCHAR(200) DEFAULT '',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 初始配置
INSERT INTO system_config (config_key, config_value, description) VALUES

```

### 表: operation_log (操作日志表)
```sql
CREATE TABLE operation_log (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    log_type VARCHAR(20) NOT NULL COMMENT '日志类型: callback-按键回调, mapping-映射操作, config-配置变更',
    device_id VARCHAR(50) DEFAULT '' COMMENT '设备ID',
    convert_code VARCHAR(100) DEFAULT '' COMMENT '转换码',
    request_data TEXT COMMENT '请求数据JSON',
    response_data TEXT COMMENT '响应数据JSON',
    status VARCHAR(20) NOT NULL COMMENT '状态: success-成功, failed-失败',
    error_message TEXT COMMENT '错误信息',
    client_ip VARCHAR(50) DEFAULT '' COMMENT '客户端IP',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_operation_log_type ON operation_log(log_type);
CREATE INDEX idx_operation_log_device_id ON operation_log(device_id);
CREATE INDEX idx_operation_log_created_at ON operation_log(created_at);
```

## API接口设计

### 1. 按键回调接口
- **路径**: `POST /api/v1/callback/button`
- **功能**: 接收终端设备按键回调
- **请求体**:
```json
{
    "cmdToken": "xxx",
    "mac": "92.92.96.62",
    "result": 1
}
```
- **响应**:
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "device_id": "92.92.96.62",
        "convert_code_sent": "0E 99 EE 22 25"
    }
}
```

### 2. 设备映射管理接口
- `GET /api/v1/mappings` - 获取所有设备映射
- `POST /api/v1/mapping` - 添加设备映射
- `PUT /api/v1/mapping/{id}` - 更新设备映射
- `DELETE /api/v1/mapping/{id}` - 删除设备映射

### 3. 系统配置接口
- `GET /api/v1/config/tcp` - 获取TCP配置
- `PUT /api/v1/config/tcp` - 更新TCP配置

### 4. 日志查询接口
- `GET /api/v1/logs` - 获取操作日志列表
  - 查询参数: `log_type`, `device_id`, `start_date`, `end_date`, `status`, `page`, `page_size`
- `GET /api/v1/logs/{id}` - 获取日志详情

## 核心逻辑流程

### 按键回调处理流程
1. 接收POST请求的按键回调数据
2. 根据回调中的`mac`字段查询数据库获取对应的`convert_code`
3. 将转换码转换为字节数据
4. 通过TCP客户端发送到配置的TCP服务器
5. 记录操作日志
6. 返回处理结果

### TCP客户端设计
- 使用`socket`库创建TCP客户端
- 支持配置TCP服务器地址和端口
- 连接失败时自动重连（可选）
- 发送数据后关闭连接
